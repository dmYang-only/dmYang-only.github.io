排序算法

发表于 2020-12-25 更新于 2021-03-20 分类于 算法 阅读次数：

前言

排序算法分类

名词解释

  • 内部排序：将需要处理的所有数据都加载到内部存储中进行排序

  • 外部排序：数据量过大，无法全部加载到内存中，需要借助外部存储进行排序

  • 稳定性：如果a原本在b前面，而a=b，排序之后a仍然在b前面

  • **In-place:**不占用额外内存

  • Out-place: 占用额外内存

  • 空间复杂度

  • 时间复杂度O(n)：1、用常数1代替运行时间中的所有加法常数

​ 2、修改只保留最高阶项

​ 3、去掉最高阶的系数

  • 常数阶O(1)<O(log 2 n)<O(n)<O(nlog2n)<O(n2)<O(n3)<O(2^n)<阶乘O(n!)

O(n):线性阶for循环

O(nlog2n):线性对数阶

O(n^2):平方阶

时间复杂度

还取决于n的规模

冒泡、选择、插入排序：O(n^2)

希尔、归并、快速排序：O(nlog2n)

基数排序：O(n*k)

冒泡排序

bubble sort 体现出一种相邻元素比较并进行交换的思想

  • 第一轮循环：相邻俩个元素交换，将大的元素放在后面；依次交换将最大的元素放在最后面

  • 第二轮循环：相邻元素交换，但交换次数少一次

思想：每俩个元素进行比较交换，大数放在后面，排序结束后最大值位于数组最后面

优化：如果发现在某趟排序中，没有发生一次交换，可以提前结束冒泡排序

时间复杂度：最佳T(n)=O(n),最差 T(n)=O(n^2)

选择排序

Select Sort 体现的是一种在余下数组中选举出最小元素的思想

外层循环进行length-1次，内层循环进行length-1次（但起始位i+1）

  • 第一次循环：遍历数组除第一个元素，直接找出最小的值，标记出该最小值的位序，与第一个元素交换。
  • 第二次循环：遍历数组（除了第一和第二个元素）直接找出最小的值，标记出该最小值的位序，与第二个元素交换。

思想：先定义第一个元素为最小数，遍历除第一个元素以外元素找出最小值的位序，与第一个元素进行交换

时间复杂度：T(n)=O(n^2)

插入排序

InsertSort体现的一种在有序区插入元素使得数组有序的思想

  • 以第二个元素开始，将插入元素在有序区进行顺序遍历比较,标记出插入位置
  • 后面元素后移
  • 将新元素放人空位

思想：分为前段有序区和后段无序区，元素在有序区中遍历比较找出插入位置，插入位置后的元素后移

希尔排序

shell Sort又称为缩小增量排序，体现在直接插入排序基础上，以增量形式进行分组，优先对远处的元素进行排序。

时间复杂度：T(n) =O(nlog2n)

思想

  • 按下标进行一定增量分组，对每组使用直接插入排序；
  • 随着增量逐渐减少，当增量减至1时，此时就成了一组

堆排序

堆排序也是一种选择排序

时间复杂度：最坏最好都是O(nlogn)

堆的概念

堆可以用数组来表示，这是因为堆是完全二叉树，而完全二叉树很容易就存储在数组中(按层序排序)

升序采用大顶堆，降序采用小顶堆

**注意:**没有要求其左右孩子的大小关系

大顶堆

每个非叶子节点的值都大于或等于其左右孩子节点的值，称为大顶堆

小顶堆

每个非叶子节点的值都小于或等于其左右孩子节点的值，称为小顶堆

堆排序

思想

  • 将排序序列构造成一个大顶堆，序列的最大值即是堆顶根节点
  • 将该最大值与序列末尾元素交换，末尾就成了最大值
  • 再将剩余n-1元素重新构成一个堆，得到n个元素的次小值。反复执行

步骤

  • 判断非叶子节点是否符合大顶堆，不符合则进行交换，最后得到一个初始大顶堆
    • 最后一个非叶子结点下标为 length/2-1
    • 判断该非叶子结点和其左右子节点大小，若父节点小于子节点，则其与较大的子节点进行交换
    • 若是父节点依旧大于子节点，则跳出循环，此时已经为大顶堆
  • 将堆顶元素和序列末尾元素进行交换
  • 对除末尾元素外，重复以上操作

快速排序

Quicksort

思想：通过一趟排序将待排记录分隔成独立的两部分，其中一部分记录的关键字均比另一部分的关键字小，则可分别对这两部分记录继续进行排序，以达到整个序列有序。

  • 从数列中挑出一个元素，称为 “基准”（pivot）
  • 重新排序数列，所有比基准值小的元素放在基准前面，所有比基准值大的元素放在基准的后面（相同的数可以到任一边），该基准就处于数列的中间位置。这称为分区（partition）操作；
  • 递归地（recursive）对小于基准值元素的子数列和大于基准值元素的子数列进行快速排序。

挖坑法

思路：

  • 定义俩个指针，一个指向起始start,一个指向结尾end，指定基准indexVal,作为’坑’

  • 依次移动左右指针,若指定坑为start，移动右指针寻找比indexVal大的值，填入坑中

​ 再移动左指针寻找比indexVal小的值，填入刚刚移动空出的位置

性能分析：原地排序，不需要辅助数组，但是递归调用需要辅助栈

最好的情况：能将数组对半分，递归调用次数才是最少。

双指针法

优化

三分单向优化

​ 对于这样一个序列 2,2,2,2,3,1，选择第一个元素2 作为 pivot 中心点，划分后得到两段子序列分别为：1和2,2,3,2，接着继续递归对子序列进行排序，对于 2,2,3,2 子序列又是将 2 作为 pivot 中心点，会发现对于这种大量元素等于 pivot 的序列单轴快排并没有起到很好的划分作用。

如果将等于 pivot 的元素也作为一个划分区段，则可以将序列划分为3段：

  • 小于 pivot 的元素
  • 等于 pivot 的元素
  • 大于 pivot 的元素。这种处理方式，会大大节省递归次数

算法思路：

  1. 由于最初红色和蓝色区域没有元素，初始化 less = left - 1，more = right + 1，cur = left。整个区间为未知部分（白色

  2. 如果当前 array[cur] < v，则 swap(array,++less,cur++)，即把红色区域向右扩大一格（less指针后移），把 array[cur] 交换到该位置，cur 指针前移判断下一个数。

  3. 如果当前 array[cur] = v，则不必交换，直接 cur++

  4. 如果当前 array[cur] > v，则 swap(array,--more,cur)，即把蓝色区域向左扩大一格（more指针前移），把 array[cur] 交换到该位置。特别注意！此时cur指针不能前移，这是因为交换到cur位置的元素来自未知区域，还需要进一步判断array[cur]。

随机边界

每次随机选取基准值，而不是固定选取左或右边界值。将随机选取的基准值和右边界值进行交换，然后就回到了之前的解法。

归并排序

分治策略

思想：合并已有序的子序列，得到完全有序的序列；

步骤：使得每一个子序列有序，再使子序列段间有序；归并排序的形式就是一棵二叉树，它需要遍历的次数就是二叉树的深度

时间复杂度：可以得出它在任何情况下时间复杂度均是O(nlogn)。

基数排序

radix sort，基数排序和桶排序相似，基数排序是桶排序的扩展，根据数组最大元素位数进行多次的桶排序，*基数排序空间复杂度为T(n) = O(n * k)*，k是指数组最大数的位数

基数排序时稳定

桶排序

桶排序，用空间换时间的经典排序算法，海量数据排序时，容易造成 OutOfMemoryError内存溢出错误，时间复杂度为T(n)=O(n+k)

桶排序

  • 取得数组中的最大数，并取得位数
  • array 为原始数组，从最低位开始取每个位组成 radix 数组
    • 将每个元素的个位数取出，看这个数应该放在哪一个桶，并按桶顺序取出元素组成数组
    • 将每个元素的十位数取出，看这个数应该放在哪一个桶，并按桶顺序取出元素组成数组
    • …….
  • 对 radix 数组进行基数排序（利用基数排序适用于小范围数的特点）
