1、注意

  • 严格区分大小写

  • 所有内容以文件形式保存，包括硬件

    • 硬盘文件/dev/sd[a-p]
    • 光盘文件/dev/sr0等

  • Linux不靠扩展名区分文件类型，靠权限。

  • Linux所有的存储设备必须挂载之后用户才能使用，包括硬盘、U盘、光盘

  • windows下程序不能直接在Linux中安装和运行

2、目录

  • /bin：
    bin 是 Binaries (二进制文件) 的缩写, 这个目录存放着最经常使用的命令。
  • /sbin：
    s 就是 Super User 的意思，是 Superuser Binaries (超级用户的二进制文件) 的缩写，这里存放的是系统管理员使用的系统管理程序。
  • /boot：
    这里存放的是启动 Linux 时使用的一些核心文件，包括一些连接文件以及镜像文件。
  • /dev ：
    dev 是 Device(设备) 的缩写, 该目录下存放的是 Linux 的外部设备，在 Linux 中访问设备的方式和访问文件的方式是相同的。
  • /etc：
    etc 是 Etcetera(等等) 的缩写,这个目录用来存放所有的系统管理所需要的配置文件和子目录。
  • /home：
    用户的主目录，在 Linux 中，每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的，如上图中的 alice、bob 和 eve
  • /srv：
    该目录存放一些服务启动之后需要提取的数据。
  • /root：
    该目录为系统管理员，也称作超级权限者的用户主目录。
  • /tmp：
    tmp 是 temporary(临时) 的缩写这个目录是用来存放一些临时文件的。

日志

/var：
var 是 variable(变量) 的缩写，这个目录中存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件

资源数据

  • /usr：
    usr 是 unix shared resources(共享资源) 的缩写，这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似于 windows 下的 program files 目录。
  • /usr/bin：
    系统用户使用的应用程序

非法关机

  • /lost+found：
    这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。

挂载

  • /media：
    linux 系统会自动识别一些设备，例如U盘、光驱等等，当识别后，Linux 会把识别的设备挂载到这个目录下。
  • /mnt：
    系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将光驱挂载在 /mnt/ 上，然后进入该目录就可以查看光驱里的内容

进程，不能写东西

/proc：
proc 是 Processes(进程) 的缩写，/proc 是一种伪文件系统（也即虚拟文件系统），存储的是当前内核运行状态的一系列特殊文件，这个目录是一个虚拟的目录，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。
这个目录的内容不在硬盘上而是在内存里，我们也可以直接修改里面的某些文件，比如可以通过下面的命令来屏蔽主机的ping命令，使别人无法ping你的机器：

/sys：

这是 Linux2.6 内核的一个很大的变化。该目录下安装了 2.6 内核中新出现的一个文件系统 sysfs 。

sysfs 文件系统集成了下面3种文件系统的信息：针对进程信息的 proc 文件系统、针对设备的 devfs 文件系统以及针对伪终端的 devpts 文件系统。

该文件系统是内核设备树的一个直观反映。

当一个内核对象被创建的时候，对应的文件和目录也在内核对象子系统中被创建。

二、常见命令

su username 更改用户

命令格式： 命令[ -选项] [参数] exp: ls -la /etc

  • 个别命令不遵循该格式
  • 有多个选项时，可以写在一起
  • 简化选项与完整选项 -a == –all

1、文件处理命令

1.1 目录查看命令

ls (list)

命令所在路径 /bin/ls 执行权限: 所有用户 功能: 显示目录文件

语法：ls 选项[-ald] [文件或目录]

  • -a 显示所有文件，包括隐藏文件
  • -l 详细信息显示
  • -d 查看目录属性

文件类型: - 二进制文件 d 目录 l软链接文件

权限: w写 r写 x执行

用户: u所有者 g所属组 o其他人

1.2 目录处理命令

可以使用相对路径，但要保证文件在当前目录下

mkdir (make directories) 所在路径：/bin/mkdir

权限: 所有用户 语法： mkdir -p [目录名]

功能：创建新目录 -p递归创建

cd (change directory) 所在路径：shell内置命令

权限：所有用户 语法：cd[目录]

功能：切换目录

pwd 功能：显示当前工作目录

cd . 当前目录 cd ..上一级目录

rmdir 所在路径 /bin/rmdir

rmdir[目录]

功能：删除空目录

cp 所在路径/bin/cp

语法：cp -rp[原文件或目录] [可复制多个] [目标目录]

​ -r 复制目录

​ -p 保留文本属性

功能：复制文件或目录

mv (move) 路径：/bin/mv

权限：所有用户 语法：mv[原文件或目录] [目标目录]

功能：剪切文件、改名

rm(remove) 路径/bin/rm

权限：所有用户 功能：删除文件或目录

语法：rm -rf[文件或目录]

​ -r 删除目录

​ -f 强制执行

1.3 文件处理命令

touch /bin/touch

权限：所有用户 语法：touch[文件名]

功能：创建空文件

  • cat /bin/cat

权限：所有用户 语法：cat[文件名]

功能：显示文件内容 -n显示行号

  • tac 功能：反向显示

  • more 语法 more[文件名]

    ​ （空格)或f 翻页

    ​ Enter 换行

    ​ q或Q 退出

    功能：分页显示文件内容

  • less 路径：/usr/bin/less

    功能：分页显示文件内容(可向上翻页pg up，/搜索内容,n看下个搜索)

  • head -n 行号 功能：前几行

  • tail -n 行号 功能：后几行

    ​ -f 功能：动态显示

1.4 链接文件

ln /bin/ln 权限：所有用户

ln -s[原文件] [目标文件] -s 创建软链接

  • 软链接：类似windows快捷方式 lrwxrwxrwx

​ 只是符号链接，箭头指向源文件。

2、权限管理命令

权限用数字表示 ：r–4(100) w–2(010) x–1(001)

2.1 改变权限

chmod 路径 /bin/chmod

语法： chmod[{ugoa}{+-=}{rwx}] [文件或目录]

​ [421] [文件或目录]

​ -R递归修改

功能：改变文件权限

目录和文件权限

file r: 可以查看文件 cat/tac/more/less/tail/head

​ w: 可以修改文件内容 vim

​ x: 可以执行文件 script command

directory r 可以列出目录内容 ls

​ w 可以目录中创建/删除文件目录 touch/rm/mkdir/rmdir

​ x 可以进入目录 cd

  • 删除文件必须在文件所在目录有w权限
2.2 改变所有者

chown

语法： chown[用户] [文件或目录]

功能：改变文件或目录的所有者

2.3 改变组名

chgrp 功能：改变组名

2.4 缺省权限

umask 语法：umask[-S]

​ -S以rwx形式显示新建文件缺省权限

​ 默认为022

功能：显示、设置文件的缺省权限

默认新建的文件不具有执行x权限

3、文件搜索命令

3.1 find

find 对系统负载高

语法：find [搜索范围] [匹配条件] [搜索内容]

功能：文件精准搜索

  • -name 根据文件名
    • -iname 不区分大小写
  • -size 根据文件大小(1数据块==512字节==0.5k)
    • +n 大于 -n小于 =等于
  • -user 根据所有者
  • -group 根据所属组
  • -type 文件类型 f文件 d目录 l链接
  • -inum 根据i节点查找

  • -amin 访问时间access
  • -cmin 文件属性change
  • -mmin 文件内容modify

find /etc -cmin -5(分钟)

逻辑搜索

  • -a 俩个条件都满足

  • -o 满足一个

搜索同时ls

  • -exec ls -l {} ;
  • -ok ls -l {} ;
3.2 其他
locate

路径 /usr/bin/locate

语法: locate 文件名 功能：在文件资料库中查找文件

ps: 资料库是定时更新的(/tmp目录下文件不会更新到资料库),updatedb更新命令

which

路径 /usr/bin/which 所有用户可以使用

功能： 搜索命令所在目录及别名信息

whereis

语法：whereis[命令名称]

功能:搜索命令所在目录及帮助文档路径

grep

路径：/bin/grep

语法：grep -iv [指定字串] [文件]

功能：在文件内搜寻字串匹配的行并输出

  • -i 不区分大小写
  • -v 排除指定字串(屏蔽所在的行)

4、帮助命令

man

路径 /usr/bin/man

语法：man [命令或配置文件] (不能加上绝对路径)

功能：获得帮助信息

  • man ls 查看ls命令的帮助信息(NAME存放信息 /选项)
  • man services 查看配置文件services的帮助信息(NAME存放信息 文件格式)
info
help

功能：获得Shell内置命令的帮助信息

passwd

6、压缩和解压命令

.zip在windows和linux都可以

6.1 gzip

路径 /bin/gzip

权限：所有用户

语法 gzip[文件]

功能：只能压缩文件，不能压缩目录，不保留原文件

压缩格式：.gz

gunzip/ gzip -d 功能:解压缩

6.2 tar

tar 路径 /bin/tar

语法： tar 选项[-zcf] [压缩后文件名] [目录]

  • -c 打包
  • -v 显示详细信息
  • -f 指定文件名
  • -z 打包同时压缩

功能：打包目录，保留原文件

文件格式：.tar.gz

tar 解包

  • -x 解包
  • -z 解压缩
  • -f 指定解压文件
  • -v 显示详细信息
6.3 zip

路径 /usr/bin/zip

语法：zip 选项[-r] [压缩后文件名] [文件或目录]

​ -r 压缩目录

语法：压缩文件或目录 压缩后格式：.zip

  • 能保留原文件

unzip 功能：解压.zip的压缩文件

6.4 bzip2

bzip2 路径： /usr/bin/bzip2

语法：bzip2 选项[-k] [文件]

​ -k 产生压缩文件后保留原文件

格式 .bz2 功能：压缩文件

bzip2 -k bodu

目录的压缩

选项：-j

  • 配合tar打包：tar -cjf Japan.tar.bz2 Japan
  • 解压：tar -xjf -k Japan.tar.bz2

bunzip2 功能：解压 .bz2文件

7、网络命令

7.1 会话

write 用户名

路径： /usr/bin/write

功能：给用户发信息，（Ctrl+D保存结束）

crtl+alt+f1 切换用户

wall 功能：发广播信息

ping 功能：测试网络连通性

语法: ping 选项 Ip地址

​ -c 指定发送次数 （ctrl+c停止）

ifconfig 路径：/sbin/ifconfig

权限：root 语法：ifconfig 网卡名称 IP地址

功能：查看和设置网卡信息

mail 路径 /bin/mail

语法：mail [用户名] 功能：查看发送电子邮件

7.2 用户

last 功能：列出目前与过去登入系统的用户

lastlog 功能：列出所有用户以及最后登录信息

  • last -u 用户uid

traceroute 功能：显示数据包到主机间的路径

netstat 语法：netstat[选项]

功能：显示网络相关信息

  • -t TCP协议
  • -u UDP协议
  • -l 监听
  • -r 路由
  • -n 显示ip地址和端口号

ps： netstat -tlun 查看本机监听的端口

​ netstat -an 查看本机所有的网络连接

​ netstat -rn 查看本机路由器

setup 权限：root

功能：配置网络

7.3 挂载命令

mount 权限：所有用户

语法：mount[-t 文件系统] 设备文件名 挂载点

类似windows分配盘符

umount 功能：卸载

8、关机重启命令

shutdown [选项] 时间

​ -c 取消前一个关机命令

​ -h 关机

​ -r 重启

系统运行级别

0 关机 1 单用户(类似windows安全模式root)

2 不完全多用户，不含NFS服务(网络文件系统(俩个linux文件共享，有安全隐患)) 3 完全多用户

4 未分配 5 图形界面 6重启

logout 退出登录
